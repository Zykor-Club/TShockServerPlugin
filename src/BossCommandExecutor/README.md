# BossCommandExecutor Boss击杀命令执行器

- **作者**: 星梦
- **版本**: v1.5.0

## 功能概述

- 当玩家击败指定的Boss后，自动执行预设的命令序列
- 支持多体节Boss判定（世界吞噬者、毁灭者、双子魔眼等），所有体节死亡后才触发命令
- 通过进度名称ID（0-22）引用Boss，简化配置流程
- 配置自动补全：reload时自动同步进度ID和Boss ID列表
- 支持首次击杀命令和常规执行命令
- 自动广播伤害排行，包含MVP标记、DPS统计等
- 浮动文本显示，支持自定义颜色和内容
- 支持占位符替换

## 指令

| 语法 | 权限 | 说明 |
|------|:----:|:-----:|
| `/bosscmd reload` | `bosscommandexecutor.reload` | 重载配置文件 |

## 占位符

| 占位符 | 说明 | 示例 |
|--------|:----:|------|
| `{boss}` | Boss名称 | "血肉墙" |
| `{boss.name}` | Boss名称（同{boss}） | "血肉墙" |
| `{boss.id}` | Boss ID | "113" |
| `{boss.type}` | Boss类型 | "113" |
| `{time}` | 当前时间 | "14:30:25" |
| `{date}` | 当前日期 | "2024-01-15" |
| `{count}` | 执行的命令数量 | "5" |
| `{exec_count}` | 累计执行次数 | "10" |
| `{world}` | 世界名称 | "新世界" |
| `{players}` | 在线玩家数量 | "5" |

### 配置项说明

#### 全局配置
| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| **插件开关** | bool | true | 控制插件是否启用 |
| **命令执行延迟(毫秒)** | int | 100 | 命令之间的执行间隔 |
| **广播执行结果** | bool | true | 是否向全服广播命令执行结果 |
| **广播消息格式** | string | 详见配置 | 广播消息的格式，支持占位符和富文本颜色 |
| **广播颜色** | object | R:50, G:205, B:50 | 广播消息的颜色（RGB格式） |
| **控制台提示** | bool | true | 是否在控制台显示命令执行成功提示 |
| **自动广播BOSS伤害排行** | bool | true | 是否自动广播Boss伤害统计信息 |

#### Boss命令配置
| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| **Boss名称** | string | 无 | Boss的显示名称 |
| **进度名称ID** | int | -1 | 进度名称ID（0-22），与Boss ID列表自动同步 |
| **Boss ID列表** | array | 无 | 该Boss对应的NPC ID数组（自动补全） |
| **需要被召唤** | bool | true | 是否只对被召唤的Boss生效 |
| **记录执行次数** | bool | true | 是否记录该Boss的命令执行次数 |
| **执行次数** | int | 0 | 当前已执行的次数（自动记录） |
| **已首次击杀** | bool | false | 是否已完成首次击杀（自动记录） |
| **首次击杀命令** | array | 无 | 首次击杀时执行的命令列表（只执行一次） |
| **常规执行命令** | array | 无 | 每次击杀都会执行的命令列表 |
| **广播结果** | bool | true | 是否广播该Boss的命令执行结果 |
| **浮动文本内容** | string | 无 | 浮动显示的文本内容，支持占位符 |
| **浮动文本颜色** | object | R:102, G:204, B:255 | 浮动文本的颜色（RGB格式） |
| **浮动文本显示时长(秒)** | int | 10 | 浮动文本显示的时间长度 |
| **启用浮动文本** | bool | true | 是否启用该浮动文本 |

### 配置自动补全机制

在reload重载配置时，插件会自动补全或修正配置项：

1. **只填写进度名称ID** → 自动补全对应的Boss ID列表
2. **只填写Boss ID列表** → 自动确定对应的进度名称ID
3. **两者都填写但不一致** → 以进度名称ID为准，更新Boss ID列表

## 插件版本

### v1.5.0
- 修复多体节Boss击杀判定：使用`BossDamageTracker.OnBossKilled`钩子确保所有体节死亡后才触发命令
- 新增进度名称ID系统：通过进度ID（0-22）引用Boss，简化配置流程
- 配置自动补全：reload时自动同步进度ID和Boss ID列表
- 精简配置结构：合并Boss命令配置和浮动文本配置为单一配置项
- 分离世吞和克脑进度：进度ID 3为世界吞噬者，进度ID 4为克苏鲁之脑
- 修复广播消息颜色：改用`player.SendMessage`支持富文本颜色格式

### v1.3.0
- 整合BOSS伤害排行功能
- 添加`自动广播BOSS伤害排行`配置项
- 优化伤害排行显示，包含队伍颜色、DPS、MVP标记

### v1.2.0
- 添加执行次数统计功能
- 支持浮动文本显示
- 添加首次击杀机制
- 支持所有泰拉瑞亚原版Boss

### v1.1.0
- 初始版本发布
- 基础Boss命令执行功能
- 支持占位符替换
